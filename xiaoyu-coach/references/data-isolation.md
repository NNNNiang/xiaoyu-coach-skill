# Data Isolation

Use these rules before reading or writing any report, memory, or public sample.

## User Classes

- Main user: personal long-term training history, body data, and progression memory.
- Named member: private member folder and member-specific history.
- Anonymous internet user: single-user folder or nickname; do not merge with main-user memory.
- Public sample: must use synthetic data or explicitly approved media.

## Rules

1. Keep each person’s videos, reports, notes, body data, and long-term observations in that person’s folder.
2. Never migrate one person’s movement fault into another person’s report.
3. If a prior report is used for comparison, confirm it belongs to the same person and same broad training category.
4. Public GitHub material must not include real videos, raw screenshots, private reports, or personal training memories unless the owner explicitly approved publication.
5. If a report discusses pain, injury, or medical risk, avoid diagnosis. Recommend stopping the risky movement and seeking qualified in-person help.

## Memory Updates

For a training system with persistent memory:

- Write long-term facts only after the report is complete and verified.
- Store workflow changes in process docs, not personal memory.
- Store member-specific changes in that member’s own memory area.
