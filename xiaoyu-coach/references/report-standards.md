# Report Standards

Use this file before generating HTML or Markdown reports.

## HTML Report

HTML is for phone-first reading. Use a stable structure:

1. Title and compact metadata
2. 总评
3. 安全优先级
4. 本次动作内容
5. 训练量与机位对比
6. 重点动作图文复盘
7. 下次训练重点
8. 教练收束

Each important movement gets one card:

- title
- 4-stage image
- movement-specific commentary
- next-session cue

Do not put all images in one section and all commentary in another.

## Markdown Report

Markdown is the detailed training-day record. Include:

- overall review
- training volume
- historical comparison
- 做对的事 vs 需改进的事
- movement notes
- red flags / safety priority
- progression conditions
- next training plan
- filming advice when needed
- conversation points

Markdown is not a plain copy of the HTML.

## Forbidden User-Facing Terms

Avoid internal workflow language in formal reports:

- MediaPipe
- OpenPose
- OpenCV
- summary
- grid
- dense
- tracking pollution
- invalid frames
- automatic judgment
- model recognition error

Use coach-facing phrasing instead:

- “这组画面主要复核整体节奏和稳定性。”
- “关键画面显示底部深度已经足够。”
- “本次不适合细抠角度，重点看可见的动作质量。”

## Progression Language

When discussing load increases, give concrete criteria:

- current load
- target reps/sets
- quality conditions
- next increment
- whether to add one top set or all sets

Example:

“60kg 先做到 8 次 x4，触胸点一致、最后一组不变形，再加 65kg x6-8 做 1 组。”
